# Deploying CryptoBot to Vercel

This guide covers deploying both the React frontend and the Express API backend to a single Vercel project.

---

## How it works

| Layer | Technology | Vercel treatment |
|---|---|---|
| Frontend | React + Vite | Built to `dist/`, served as a static site |
| Backend | Express.js | Runs as a serverless function via `api/index.js` |

All `/api/*` requests are routed to the serverless function automatically. Everything else is served from the static build and falls back to `index.html` for client-side routing.

---

## Prerequisites

- A [Vercel account](https://vercel.com/signup) (free tier is sufficient)
- Your project pushed to a Git repository (GitHub, GitLab, or Bitbucket)
- [Vercel CLI](https://vercel.com/docs/cli) installed (optional, for CLI deployment)

```bash
npm install -g vercel
```

---

## Environment Variables

You must configure these in the Vercel dashboard under **Project → Settings → Environment Variables**. Do **not** commit your `.env` file — it is excluded by `.gitignore`.

### Required

| Variable | Used by | Description |
|---|---|---|
| `ANTHROPIC_API_KEY` | Backend (serverless) | Powers the AI analysis endpoint |

### Optional but recommended

| Variable | Used by | Description |
|---|---|---|
| `VITE_FRED_API_KEY` | Frontend (browser) | Fetches live DXY data from FRED. Must have the `VITE_` prefix. |

> **Important:** Vite only exposes environment variables prefixed with `VITE_` to the browser bundle. Any variable without this prefix is server-side only. In your local `.env` file, rename `FRED_API_KEY` → `VITE_FRED_API_KEY` for the DXY feature to work.

## Option 1 — Deploy via the Vercel Dashboard (recommended)

1. Push your project to GitHub (or another Git provider).
2. Go to [vercel.com/new](https://vercel.com/new) and import your repository.
3. Vercel will auto-detect the project. Confirm or set these settings:
   - **Framework Preset:** Vite
   - **Build Command:** `npm run build`
   - **Output Directory:** `dist`
   - **Install Command:** `npm install`
4. Add all required environment variables (see table above).
5. Click **Deploy**.

Vercel reads `vercel.json` automatically, so routing is configured without any extra steps.

---

## Option 2 — Deploy via the Vercel CLI

```bash
# Log in
vercel login

# Deploy (follow the prompts)
vercel

# Set environment variables
vercel env add ANTHROPIC_API_KEY
vercel env add VITE_FRED_API_KEY

# Promote to production
vercel --prod
```

## Build Commands Reference

| Command | Purpose |
|---|---|
| `npm install` | Install all dependencies |
| `npm run build` | Compile TypeScript and bundle the frontend with Vite → `dist/` |
| `npm run dev` | Start both frontend and backend locally for development |
| `npm run client` | Start only the Vite dev server (port 5000) |
| `npm run server` | Start only the Express server (port 3001) |

## Project Structure on Vercel

```
/
├── api/
│   └── index.js          ← Vercel serverless function (Express app)
├── dist/                 ← Built frontend (generated by npm run build)
├── server/
│   ├── app.js            ← Express app (shared between local & serverless)
│   ├── index.js          ← Local dev entry (calls app.listen)
│   └── routes/
│       ├── aiProxy.js    ← POST /api/analyze → Anthropic AI
│       └── proxy.js      ← GET  /api/health
├── src/                  ← React frontend source
├── vercel.json           ← Routing rules
└── package.json
```

## How routing works in production

`vercel.json` defines two rewrite rules:

```json
{ "source": "/api/:path*", "destination": "/api/index" }
```
All API calls from the frontend (e.g. `POST /api/analyze`) are forwarded to the serverless Express function.

```json
{ "source": "/(.*)", "destination": "/index.html" }
```
All other routes return `index.html`, enabling React Router to handle client-side navigation.

## Verifying the deployment

Once deployed, check the following endpoints to confirm both layers are working:

```
GET  https://your-app.vercel.app/api/health   → { "status": "ok", ... }
POST https://your-app.vercel.app/api/analyze  → AI analysis JSON
```

If `GET /api/health` returns `{ "status": "ok" }`, the backend serverless function is operational. If the AI analysis endpoint returns a result, your `ANTHROPIC_API_KEY` is correctly configured.

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| `/api/analyze` returns 500 with "ANTHROPIC_API_KEY not configured" | Missing env var | Add `ANTHROPIC_API_KEY` in Vercel → Settings → Environment Variables |
| DXY always shows fallback value | `VITE_FRED_API_KEY` missing or wrong prefix | Set `VITE_FRED_API_KEY` (not `FRED_API_KEY`) in Vercel env vars |
| Blank page after navigating directly to a route | SPA fallback missing | Verify `vercel.json` is present and contains the `/(.*) → /index.html` rewrite |
| "Module not found" errors during build | Dependency missing | Run `npm install` locally, confirm `package-lock.json` is committed |
